    Documentation Overview

    This documentation is written using Material for MkDocs. This is a wrapper for MkDocs that makes it easier to make the site look better and be more readable

    It is hosted on GitHub and should be relativly straightforward to edit and add things to

    MkDocs Commands

    to get started, open visual studio code, navigate to the right folder, and start a new termianl. The terminal commands are listed below in order for convenience

    1. python -m venv venv — Start the virtual environment
    2. .\venv\scripts\activate — Activate the virtual environment
    3. optional if needed — Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass — Bypass the system to activate environment
    4. pip install mkdocs-material — Install Material for MkDocs
    5. mkdocs build — Build the documentation site

    If you save in vs code while the site is running it will auto update

    Add a New Page

    to add a new page to the documentation, navigate to the correct folder and add a new file called "your_file_name.md." The position in the folder structure technically doesn't matter but it is useful for organization.

    In order to place that code in the correct place on the page, navigate to the mkdocs.yml file. Find the nav tab as seen below and update it to tell the system where to place your page

    nav:
- Documentation:
    - Overview: documentation/documentation.md
- Actuators:
    - Overview: actuators/actuators-overview.md
    - Harmonic Drive: actuators/harmonic-drive.md

    Adding a new folder/topic works the same way. You should add a new folder for organization and update the .yml to place make a new tab in the documentation

    Editing your Page

    most AI systems are fairly good at working with material for mkdocs, just make sure you tell them specifically "material for mkdocs" and not just mkdocs.

    On the Material for MkDocs website under reference there is also a lot of cood information about how to make the site do what you want. The getting started tab also has some information, but most of that stuff was handeled by me when I made the rest of this documentation, so you shouldn't have to worry about that too much
